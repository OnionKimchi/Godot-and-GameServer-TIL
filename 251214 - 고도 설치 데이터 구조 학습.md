### 🧠 배운 개념 정리 (핵심)

#### 1. Godot의 데이터 단위 – Resource

* Godot의 `Resource`는 **Unity의 ScriptableObject와 거의 1:1 대응**
* 공통점:

  * 에디터에서 데이터 에셋으로 관리
  * 코드와 데이터 분리
  * 여러 객체에서 참조 가능
* 차이점:

  * Resource는 **씬, 노드, 설정 전반에 걸쳐 더 광범위하게 사용됨**
  * “특수한 데이터 컨테이너”라기보다 **엔진 전반의 기본 데이터 포맷**

👉 느낌적으로
Unity: *“SO는 패턴”*
Godot: *“Resource는 기본값”*

---

#### 2. Scene / Node 구조와 데이터의 관계

* Scene은 단순 레벨이 아니라 **재사용 가능한 객체 단위**
* Node는 GameObject + Component를 합쳐놓은 느낌
* 데이터(Resource)는:

  * Node에 직접 붙거나
  * Scene 내부에서 참조되거나
  * 순수 데이터 에셋으로 독립 존재 가능

→ **데이터 → 구조 → 인스턴스** 흐름이 명확함

---

#### 3. Unity 개발자 관점에서의 첫 인상

* Godot은 “이건 이렇게 쓰세요”가 아니라
  **“기본 재료를 줄 테니 구조는 네가 짜라”** 쪽에 가까움
* SO 남용으로 데이터 구조가 복잡해졌던 Unity 경험을 떠올리면
  Godot은 애초에 데이터 설계를 전제로 한 엔진처럼 느껴짐

### 🔁 Unity와 비교하며 정리

| 항목     | Unity                  | Godot       |
| ------ | ---------------------- | ----------- |
| 데이터 에셋 | ScriptableObject       | Resource    |
| 기본 단위  | GameObject + Component | Node        |
| 씬 개념   | 레벨 중심                  | 재사용 오브젝트    |
| 데이터 사용 | 선택적 패턴                 | 엔진 전반 기본 구조 |
| 느낌     | 기능 중심                  | 구조 중심       |
