### TIL – 바이브 코딩에서 .md의 힘

프로젝트에 `.md` 문서를 잘 설계해 두면
VS Code 같은 IDE에 내장된 AI(코파일럿)가 **코드를 더 맥락에 맞게 생성한다**는 걸 배웠다.

* IDE AI는 모든 코드를 다 읽는 게 아니라
  **현재 파일 + 주변 파일 + 문서 패턴**을 컨텍스트로 삼는다.
* 그래서 `게임_룰북.md`, `ToDo.md`, `답변지시사항.md` 같은 문서는
  사실상 **상시 프롬프트**처럼 작동한다.
* 실제로 예전에 프로젝트에 있던 AI 기자용 프롬프트 문서 때문에
  코파일럿이 답변 톤까지 흉내 내는 현상을 겪었다.
* 결론적으로, **IDE에 프롬프트를 직접 설정하지 않아도**
  프로젝트 구조와 `.md` 문서 자체를 프롬프트처럼 설계할 수 있다.

👉 바이브 코딩의 핵심은
“AI에게 지시를 매번 말하는 것”이 아니라
**AI가 숨 쉬는 프로젝트 환경을 만들어주는 것**이다.
